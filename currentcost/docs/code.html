Modules¶

This project contains 2 mains modules:
  • Utils contains all useful functions and Variables for this project,
  • RabbitMQ_messager send message to RabbitMQ or display it in stdout.

Utils¶

Globals and function needs for this project.

currentcost.utils.argument_parser()[source]¶

Method that parse arguments from command line, return an error in case of missing parameter or return arguments with their value.

Returns:dict – Dict containing arguments.
currentcost.utils.data_validator(data, variable_name, site_name)[source]¶

Analyse data from currentcost and return according TOPIC and MESSAGE.

Parameters:
  • data – XML string that contain data.
  • variable_name (str.) – Name of the Variable.
  • site_name (str.) – Name of the Site.
Returns:

str – Topic of the message (error or success) and Message containing error description or data sent by CC.

Create log message starting current cost.

Parameters:
  • variable_name (str.) – Name of the Variable.
  • site_name (str.) – Name of the Site.
  • tty_port (str.) – TTY port path.

Active verbose mode.

RabbitMQ_messager¶

Method that send message on RabbitMQ or display it on stdout.

class currentcost.messager.RabbitMQMessager(username, password, host)[source]¶

This class send message to RabbitMQ or in stdout following user choice.

consume(topic, callback)[source]¶

Method that wait for a message on RabbitMQ topics channels.

Parameters:
  • topic (str.) – Channel where to send message (RabbitMQ filter).
  • callback (Function.) – Function that compute result of message (async).

  • message (str.) – Message to send to RabbitMQ.
  • out (StringIO.) – Stdout redirection in case of test (default: stdout).
